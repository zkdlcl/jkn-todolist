# 문서화 체크리스트

## 문서화 유지 관리 가이드

이 문서는 JKN-TODOLIST 프로젝트에서 문서화를 유지 관리하는 데 사용되는 체크리스트입니다. 새로운 기능을 추가하거나 기존 기능을 업데이트할 때마다 아래 항목들을 확인하세요.

---

## 🔁 정기 문서화 점검 체크리스트

### 1. 문서 업데이트 (필수)

- [ ] `docs/7-implementation_plan.md`에 새로운 개발 단계/기능 추가 또는 기존 단계 업데이트
- [ ] `README.md` (루트)에 기능 추가 사항 반영
- [ ] `docs/README.md` (문서 인덱스)에 새 문서 반영
- [ ] `docs/extentions/README.md`에 확장 기능 문서 구조 반영

### 2. API 문서 업데이트 (필요 시)

- [ ] `swagger/swagger.json`에 새 엔드포인트 추가
- [ ] `docs/API/`에 관련 API 문서 업데이트 또는 추가
- [ ] API 변경 사항에 대한 테스트 가이드 업데이트

### 3. 확장 기능 문서화 (확장 기능 추가 시)

- [ ] `docs/extentions/`에 통합 기능 명세서 생성 (@--- 마커 기반 구조 사용)
- [ ] 관련 기능에 대한 사용자 시나리오 작성
- [ ] 기술 명세, 백엔드/프론트엔드 구현 가이드 포함
- [ ] 확장 가능한 아키텍처 설계 고려 사항 포함
- [ ] 구현 가이드 및 시작 방법 포함

### 4. 파일 정리

- [ ] 관련 없는 임시 파일이나 디버깅 파일 제거
- [ ] 불필요한 주석 및 중복 정보 제거
- [ ] 문서 파일 구조 검토 및 정리 (MVP, 확장 기능 구분)

### 5. 테스트 및 검증

- [ ] 문서에 명시된 모든 API 엔드포인트가 실제로 동작하는지 확인
- [ ] 설치 및 설정 가이드를 따라 실제 환경에서 테스트
- [ ] 사용자 시나리오에 따라 실제 기능이 동작하는지 확인

## 📋 @--- 마커 기반 문서화 가이드

### 통합 문서 구조 (@--- 마커 사용)

기능 관련 모든 정보를 하나의 문서에 @--- 마커를 사용하여 구조화하세요:

- [ ] `@FeatureName-Overview` - 기능 개요
- [ ] `@FeatureName-UserScenarios` - 사용자 시나리오
- [ ] `@FeatureName-Requirements` - 기능 요구사항
- [ ] `@FeatureName-Frontend` - 프론트엔드 구현 정보
- [ ] `@FeatureName-Backend` - 백엔드 구현 정보
- [ ] `@FeatureName-API` - API 명세
- [ ] `@FeatureName-UIUX` - UI/UX 설계
- [ ] `@FeatureName-Database` - 데이터베이스 정보
- [ ] `@FeatureName-Extensibility` - 확장성 고려 사항
- [ ] `@FeatureName-Implementation-Guidelines` - 구현 가이드
- [ ] `@FeatureName-Acceptance-Criteria` - 인수 조건

### 문서 접근 방식

- [ ] @--- 마커를 사용하여 관련 정보를 빠르게 찾을 수 있도록 함
- [ ] IDE에서 `Ctrl+Shift+F` 또는 `Cmd+Shift+F`를 사용하여 마커 검색
- [ ] 여러 파일에 흩어진 정보를 통합 문서에서 한눈에 확인
